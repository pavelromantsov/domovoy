# DeviceType (тип устройства)

| Name | Description |
| :--- | :--- |
| **Sensor** | **Сенсор** — датчик, который собирает данные из внешней среды (температура, движение, протечка, освещенность) и передает их в систему |
| **Actuator** | **Актуатор** — исполнительные устройства (умные лампочки, диммеры, светодиодные ленты, умные розетки, реле, силовые контакторы, термоголовки на радиаторы, контроллеры кондиционеров, теплого пола, приводы штор/жалюзи/ворот/заслонок, электрозамки, запорные краны для воды/газа) |
| **Controller** | **Интерфейс** — кнопка, сенсорный экран, умная колонка, приложение |

# DeviceCategory (категория устройства)

| Name | Description |
| :--- | :--- |
| **Security** | **Безопасность** — следят за целостностью периметра и несанкционированным доступом (вторжение, кража) |
| **Climate** | **Климат** — мониторят комфорт внутри дома и состояние окружающей среды (отопление, вентиляция/бризеры, увлажнение, освещение) |
| **Emergency** | **Предотвращение аварий** — следят за внутренними угрозами, способными нанести ущерб имуществу или здоровью (затопление, пожар, отравление газом) |
| **Energy** | **Энергопотребление** — контроль расхода электричества, воды, газа |

# SensorRole (роль сенсора)

| Name | Description |
| :--- | :--- |
| **EndDevice** | Шлет в хаб/шину только свои данные |
| **Repeater** | Шлет в хаб/шину свои данные, а также умеет пересылать данные от других датчиков |

# SensorSettings (описание параметров сенсора)


| Name | Description | Пример |
| :--- | :--- | :--- |
| **Sensitivity** | Чувствительность — способность датчика замечать минимальные изменения | При изменении чувствительности датчика движения можно исключить срабатывание на домашних животных |
| **Threshold** | Порог срабатывания — конкретное значение, при котором датчик говорит, что событие произошло | Термостат включает обогрев на 21.5°C |
| **BlindTime** | Время нечувствительности — время, в течение которого датчик «спит» после срабатывания, чтобы не спамить сигналами и беречь батарейку | Человек входит в комнату, датчик движения его «замечает», переходит в статус «Движение». В следующие 60-90 секунд ничего не отслеживает |
| **HoldTime** | Время удержания — время, в течение которого датчик считает, что событие всё еще длится | Датчик присутствия «удерживает» статус наличия в комнате, даже если вы замерли и читаете книгу. Обычный датчик движения (PIR) в такой ситуации решит, что в комнате никого нет, и выключит свет |
| **PowerSupply** | Тип питания | |
| **ReportingInterval** | Интервал отчета — частота, с которой датчик отправляет данные на хаб/шину | Если термостат шлет данные раз в 30 минут, вы не успеете вовремя выключить обогреватель, и в комнате станет жарко |
| **SamplingRate** | Частота дискретизации — число, с которой сенсор считывает данные из среды | Датчик может замерять температуру каждую секунду, но отправлять отчет раз в 5 минут. Это позволяет ему мгновенно среагировать на резкий скачок (например, открытие окна), не дожидаясь планового отчета |
| **Hysteresis** | Гистерезис — разница между значением включения и выключения | Вы настроили включение увлажнителя на 40% влажности. Если гистерезис равен 0, увлажнитель будет включаться на 39.9% и выключаться на 40.1% каждые 5 секунд. Это убьет мотор увлажнителя |
| **DeadBand** | Зона нечувствительности — диапазон значений вокруг текущего показания, внутри которого датчик игнорирует изменения | Гистерезис нужен для логики включения/выключения, а Deadband — чтобы датчик не отправлял отчеты, если ваша температура изменилась всего на 0.01°C (шум) |
| **FieldOfView** | Зона обзора — угол обзора датчика | |
| **OperatingRange** | Рабочий диапазон — границы температуры, влажности и т.п., при которых датчик работает корректно | Бытовые датчики часто выходят из строя или быстро разряжаются на морозе (в неотапливаемом гараже) или при высокой влажности (в ванной бане) |
| **Accuracy** | Погрешность — насколько данные соответствуют реальности | Дешевые датчики температуры могут ошибаться на 1–2 градуса. Для комфорта это терпимо, но для инкубатора или варки пива — критично |
| **Resolution** | Шаг измерения — минимально возможная разница, которую датчик технически способен зафиксировать | Датчик температуры может иметь Accuracy ±0.5°C, но его Resolution может быть 0.1°C. Это значит, что он может показать 22.1, 22.2, 22.3, а не просто прислать целое градус |
| **Latency** | Задержка сигнала — время от момента физического события до момента, когда контроллер (хаб) получил сигнал | Если вы используете датчик для включения света, задержка более 200 мс будет ощущаться как «тормознутость» системы |
| **Calibration** | Поверка/калибровка — программная подстройка данных для устранения накопленной измерительной ошибки | Датчики со временем «врут» (отравление химических сенсоров). Калибровка заменяет функцию самокоррекции |
| **KeepAliveInterval** | Интервал проверки связи — время, через которое датчик присылает короткий сигнал «я в сети», даже если ничего не изменилось | Если датчик не прислал сигнал в течение нескольких таких периодов, система пометит его как «недоступный» или «умер», и поставит задачу вам на проверку или выдаст аварию |
| **WarmUpTime** | Время прогрева — время, необходимое сенсору после включения питания для выхода на рабочий режим | Датчики качества воздуха (CO2, VOC) или горючих газов требуют от 30 секунд до нескольких минут «прогрева» химического элемента, прежде чем их показаниям можно будет верить |
| **CalibrationOffset** | Смещение калибровки — программная правка показаний (индекс) | Если ваш датчик системно завышает температуру на 1.2 градуса по сравнению с эталонным термометром, вы вводите Offset -1.2, чтобы система видела корректные данные |
| **BatteryLevelReporting** | Индикация заряда — способность датчика передавать точный процент остатка заряда батареи или критический статус «Low Battery» | |
| **DirectBinding** | Прямая привязка — способность датчика отдавать команду напрямую актуатору, минуя хаб | Датчик движения дает команду лампочке включиться. Даже если хаб упал, свет все равно включится |
| **PulseWeight** | Вес импульса, полученного от обычного (не смарт) устройства | Определяет, сколько литров соответствуют одному импульсу, полученному от обычного («глупого») счетчика воды |
